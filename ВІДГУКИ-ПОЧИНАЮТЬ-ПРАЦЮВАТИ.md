# Чому відгуки не зберігаються і як це виправити (30 секунд)

Код відгуків готовий і правильний. Проблема **тільки** в правилах доступу Firebase —
вони поки що не дозволяють зберігати дані в колекцію `reviews` (і, можливо, у `bookings`).

Це налаштування в **Firebase Console**, його не можна змінити з коду без твого логіну.

## Як виправити (найпростіший спосіб — через сайт Firebase)

1. Відкрий: https://console.firebase.google.com/project/boxing-booking-17e44/firestore/rules
2. Зітри все, що там є, і встав ось це:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /bookings/{doc} {
      allow read, create, delete: if true;
      allow update: if false;
    }
    match /reviews/{doc} {
      allow read, create, update, delete: if true;
    }
    match /settings/{doc} {
      allow read, write: if true;
    }
  }
}
```

> ⚠️ Якщо ти вже публікував старі правила раніше — **онови їх на ці**. Тут у `reviews`
> додано `update` (потрібно, щоб у адмін-панелі працювала зміна порядку відгуків ▲▼).

3. Натисни синю кнопку **Publish** (Опублікувати).

Готово. Тепер і записи, і відгуки зберігатимуться.

## Перевірка

Відкрий сайт → «Залишити відгук» → постав зірочки, ім'я, текст → «Надіслати».
Має з'явитись «✓ Дякуємо за відгук!» і відгук одразу покажеться вгорі списку.

## Примітка про безпеку

Ці правила відкриті (будь-хто може створювати/видаляти записи) — так само, як працює
форма запису зараз. Для маленького персонального сайту це нормально. Якщо згодом захочеш
надійніше — додамо вхід тренера через Firebase Auth і закриємо видалення лише для нього.
(Ті самі правила лежать у файлі `firestore.rules` — для деплою через Firebase CLI.)
