## Оркестратор / Sub-agents для цього проєкту

Якщо робота над сайтом ведеться через Claude Code з кількома агентами — розподіл ролей:

### 🎯 Оркестратор
**Обов'язки:**
- Приймає задачу від власника (зазвичай "Етап N" з плану нижче)
- Розбиває задачу на підзадачі для потрібних sub-agents
- Передає кожному agent тільки релевантний контекст (дизайн-систему — Дизайнеру, Firebase config — Розробнику, і т.д.)
- Після виконання — перевіряє результат всіх agents разом
- Звітує власнику простою мовою: що зроблено, що перевірено, що далі

### 💻 [РОЗРОБНИК]
**Обов'язки:**
- Пише та редагує код: HTML/CSS/JS
- Реалізує Firebase-інтеграцію (форма запису → Firestore, за конфігом вище)
- Верстає секції за описом з плану етапів
- Підключає зовнішні елементи: Google Maps embed, посилання на Telegram/Instagram
- Не приймає дизайн-рішень самостійно — узгоджує з [ДИЗАЙНЕР]

### 🎨 [ДИЗАЙНЕР]
**Обов'язки:**
- Контролює відповідність дизайн-системі (платина+графіт, Inter+Playfair, marquee, лого, анти-флеш фон)
- Перевіряє кожен новий блок: кольори, шрифти, відступи, адаптивність (мобільна версія)
- Слідкує за забороною на чужі фото боксерів (тільки власні/плейсхолдери до заміни)
- Пропонує правки, якщо новий елемент "випадає" зі стилю

### ✅ [ТЕСТЕР]
**Обов'язки:**
- Перевіряє, що форма коректно зберігає дані у Firestore
- Перевіряє адаптивність (десктоп/мобільний)
- Перевіряє всі посилання (Telegram, Instagram, карта, якорі навігації)
- Перевіряє, чи сайт коректно відкривається на GitHub Pages після деплою
- Звітує про знайдені баги Оркестратору

**Принцип роботи:** Оркестратор формулює задачу для кожного sub-agent з посиланням на відповідний розділ цього файлу. Sub-agent виконує вузьку задачу, звітує оркестратору. Оркестратор збирає результат і повідомляє власнику підсумок простою мовою.

Тепер працює і локально (подвійний клік), і на https.
